# 产品需求文档 (PRD)：个人作品集网站

## 1. 项目概述

### 1.1 背景与目标

本项目旨在为一名后端开发工程师打造一个现代化的个人作品集网站。该网站不仅是个人技术成果的沉淀，也是对外界展示专业技能和项目经验的核心窗口。

### 1.2 目标用户

- **招聘方/合作伙伴**：了解开发者的技术栈、项目质量和审美品味。
- **开发者本人**：记录成长历程，作为个人品牌的线上名片。

---

## 2. 设计规范 (根据调研发现)

### 2.1 整体风格

- **清爽简约**：避免冗余信息，留白适度，追求极简主义。
- **高级感/现代化**：使用平滑的过渡动画、高质量的排版以及精致的阴影效果。

### 2.2 色彩体系

- **主色调**：以 **蓝绿色 (Teal/Aqua)** 为主，传达专业、稳重且不失活力的感官。
- **辅助色**：建议搭配深灰/冷黑作为背景或文字颜色，辅以柔和的浅灰色背景块。

---

## 3. 功能需求

### 3.1 首页 (Hero Section)

- **大标题 (Headline)**：醒目的 Slogan，例如“专注于高效后端架构的开发者”。
- **个人简介 (Sub-headline)**：简短的一两句话，概括核心价值。
- **形象展示 (Visual)**：一张高质量的能融入整体风格的风景图片或极简风格的插画/3D 元素。
- **行动呼吁 (CTA)**：例如“查看我的项目”或“联系我”的按钮。

### 3.2 个人介绍 (About Me)

- **详细介绍**：描述职业背景、技术热情以及解决问题的思路。
- **技能列表 (Skills)**：
  - 采用标签云或图标卡片形式。
  - 区分核心技能（如：Java, Go, Spring Boot, K8s）和辅助技能。

### 3.3 项目展示 (Project Showcase)

- **展示形式**：响应式卡片列表。
- **项目卡片内容**：
  - **项目截图**：高分辨率、比例统一的预览图。
  - **项目名称**：清晰的标题。
  - **简短描述**：一两句话说明项目用途。
  - **技术栈标签**：展示该项目使用的关键技术（如：Redis, MySQL, Docker）。
  - **项目链接**：提供 GitHub 源码链接或在线 Demo 访问按钮。

### 3.4 联系方式 (Contact)

- **快捷链接**：
  - **邮箱**：支持点击调起邮件客户端。
  - **GitHub**：直接跳转至个人主页。
  - **社交媒体**：如 LinkedIn、知乎或博客地址。

### 3.5 扩展功能 (Extended Features)

- **深浅色模式切换**：支持用户在导航栏切换“浅色/深色”模式。
- **访问量统计**：在页面底部（Footer）展示全站的访问次数。
- **留言板**：提供一个简单的区域，供访问者留下意见或反馈（后端开发可根据需求选择第三方集成或自研简单接口）。



---

## 4. 交互与体验要求 (UX)

1. **响应式设计**：完美适配 PC、平板和手机移动端。
2. **平滑滚动**：点击导航栏时，页面应平滑滚动到对应模块。
3. **悬浮特效**：项目卡片在鼠标悬停时应有轻微放大或阴影加深的效果，增强互动感。
4. **暗黑模式 (可选建议)**：考虑到后端开发的习惯，支持深浅色模式切换能进一步提升“高级感”。

---

## 5. 技术建议 (供参考)

由于你是后端开发，建议前端可以采用以下轻量且现代的方案：

- **框架**：Next.js 或 Vite + React/Vue3。
- **样式**：Tailwind CSS (非常适合快速实现蓝绿色系和清爽风格)。
- **部署**：Vercel 或 GitHub Pages。
